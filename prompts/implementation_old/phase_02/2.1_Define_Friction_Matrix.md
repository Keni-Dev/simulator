# Phase 2.1: Define Triboelectric Materials & Friction Matrix

**Phase:** 2 - Core Physics State & Data Layer
**Step:** 2.1

**Context:**
TriboSim simulates literal physics. This file acts as the database of materials and the $k$ (friction coefficient) matrix that drives the math engine ($q = k \cdot v_{rel} \cdot \Delta t \cdot (T_A - T_B)$).

**Skills to Load:**
- `research-engineer` — Ensures physics logic and constants are modeled accurately for web simulation.

**Instructions:**

1. **Create `src/constants/triboelectric.ts`:**
   - Define a constant `MATERIALS` of type `Record<string, Material>` (import `Material` from `types/physics`).
   - Add at least these 5 core materials:
     - `glass`: Highly positive (`triboIndex: +5`).
     - `nylon`: Positive (`triboIndex: +3`).
     - `cotton`: Neutral (`triboIndex: +0.5`).
     - `silk`: Negative (`triboIndex: -2`).
     - `teflon`: Highly negative (`triboIndex: -5`).
     - `copper`: Conductor (`triboIndex: 0`, `type: 'conductor'`).

2. **Build the Friction Matrix:**
   - Define a constant `FRICTION_MATRIX` (type `Record<string, number>`).
   - Create key pairs (e.g., `'glass-silk': 1.5`, `'teflon-nylon': 2.0`). Feel free to estimate these scaling numbers to ensure the UI charges up at a satisfying rate when rubbed.

3. **Export a Helper Function:**
   - Create and export `getFrictionCoefficient(idA: string, idB: string): number`.
   - This function must sort `idA` and `idB` alphabetically so that querying `silk-glass` returns the same value as `glass-silk`.
   - Fall back to a default multiplier (e.g., `1.0`) if a specific pair isn't in the matrix.

**Verification:**
- The file exports `MATERIALS` and `getFrictionCoefficient` cleanly.
- `getFrictionCoefficient('glass', 'teflon')` === `getFrictionCoefficient('teflon', 'glass')`.
