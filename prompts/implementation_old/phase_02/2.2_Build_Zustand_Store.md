# Phase 2.2: Build the Global Zustand Physics Store

**Phase:** 2 - Core Physics State & Data Layer
**Step:** 2.2

**Context:**
The core physics calculations run continuously inside `useFrame` at 60fps. Using Zustand instead of generic React Context is hyper-critical because we cannot afford to trigger DOM re-renders every single frame for physics updates.

**Skills to Load:**
- `cc-skill-frontend-patterns` — For correct and performant Zustand store instantiation.
- `clean-code` — Avoiding bloated state logic. Ensure actions are concise.

**Instructions:**

1. **Create `src/store/useSimulationStore.ts`:**
   - Import `create` from `zustand`.
   - Import `getFrictionCoefficient` and `MATERIALS` from `src/constants/triboelectric.ts`.

2. **Define the State Interface (`SimulationState`):**
   - **Selections:** `leftMaterialId` (default `'teflon'`), `rightMaterialId` (default `'glass'`).
   - **Physics Data:** `q1` (default `0`), `q2` (default `0`), `qNet` (computed).
   - **Positional Data:** `leftHandPos: [number, number, number]`, `rightHandPos: [number, number, number]`.

3. **Define Actions within the Store:**
   - `setMaterial(hand: 'left' | 'right', materialId: string)`: Updates either left or right selection.
   - `updateHandPos(hand: 'left' | 'right', pos: [number, number, number])`: Updates specific hand coordinates.
   - `groundCharges()`: Hard resets both `q1` and `q2` to `0`. Overwriting any existing charge.
   - **The Physics Action - `applyFriction(v_rel: number, deltaTime: number)`:**
     - Get current state: `get()`.
     - Fetch materials: `matA = MATERIALS[state.leftMaterialId]`, `matB = MATERIALS[state.rightMaterialId]`.
     - *Conductor Bailout:* If either material has `type === 'conductor'`, auto-trigger `groundCharges()` and return. Conductors instantly dissipate static charge.
     - Lookup Friction: `k = getFrictionCoefficient(matA.id, matB.id)`.
     - Math: `deltaQ = k * v_rel * deltaTime * (matA.triboIndex - matB.triboIndex) * 0.1` (the `0.1` is a scaling factor, adjust as needed so charges don't hit infinity instantly).
     - Set state: `set({ q1: state.q1 - deltaQ, q2: state.q2 + deltaQ, qNet: (state.q1 - deltaQ) + (state.q2 + deltaQ) })`.

**Verification:**
- The Zustand store executes `applyFriction` mathematically perfectly.
- `qNet` functionally stays absolutely zero modulo floating point rounding errors.
- Selecting 'copper' instantly drains existing charge.
