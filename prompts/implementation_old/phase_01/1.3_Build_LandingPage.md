# Phase 1.3: Build the Landing Page

**Phase:** 1 - Landing Page & Setup Screen
**Step:** 1.3

**Context:**
Replaces the default Next.js index with the actual TriboSim entry point. Demands attention with heavy typography, displays basic instructions, and contains the critical "Enable Camera" CTA button.

**Skills to Load:**
- `frontend-design` — Strict adherence to the `01_landing_setup.md` UI layout and glow mechanics.

**Instructions:**

1. **Modify `src/app/page.tsx`:**
   - This must be a Client Component (`"use client"`) if you plan to use `useRouter()` logic.
   - Wipe out the default NextJS boilerplate.
   - Render the custom `<BackgroundMesh />` as the first sibling.

2. **Build the NavBar layer:**
   - Fixed to the top, height 64px, 1px bottom border (`border-zinc-800`).
   - Backdrop filter applied: `backdrop-blur-md bg-black/40`.
   - Content: Logo (L)/ "Ver 1.0" (Center)/ Github (R).

3. **Build the Hero Layout:**
   - Centered flex column with `max-w-4xl`.
   - Title: `font-black text-6xl tracking-tight`. Text value: "Visualize Net Charge in Real-Time."
   - Subtitle below: `text-zinc-400 text-lg`.

4. **Build the Setup CTA Card:**
   - Render a centered card bordered by `#2A2A2D`.
   - Add a `lucide-react` Camera Icon inside.
   - Button Element styling is strictly critical:
     - `bg-neon-cyan` background.
     - `text-black font-bold`.
     - Hover State: Apply a massive neon cyan drop shadow: `hover:shadow-[0_0_20px_rgba(0,229,255,0.4)]` and `transition-all`.
     - Click Logic: Setup a fake router push (`onClick={() => router.push('/simulator')}`) for now.

5. **Build the Instructions Grid:**
   - Render a 3-column CSS grid below the CTA explaining the workflow: "Enable Camera", "Select Materials", "Rub Hands".

**Verification:**
- Page routes correctly to `/`.
- The CTA button aggressively glows cyan when hovered.
- The layout matches the scientific, high-end feel.
