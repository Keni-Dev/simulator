# Phase 4.2: Physics Loop & Collision Detection

**Phase:** 4 - Computer Vision (MediaPipe) Integration
**Step:** 4.2

**Context:**
Now that Zustand has real-time hand coordinates, the 3D simulation must constantly check if the hands are colliding and calculate their relative velocity ($v_{rel}$) to drive the math engine.

**Skills to Load:**
- `research-engineer` — For precise geometric mathematics and delta-time calculations within a React Three Fiber `useFrame` loop.

**Instructions:**

1. **Modify `src/components/scene/SimulationCanvas.tsx`:**
   - Import `useFrame` from `@react-three/fiber`.

2. **Implement the Collision & Math Loop (`useFrame`):**
   - Note: Do NOT use `useSimulationStore()` standard hook inside `useFrame` to read variables, it causes re-renders. Use `useSimulationStore.getState()`.
   - Track previous positions using a `useRef` (e.g., `prevLeftPos`, `prevRightPos`).
   - Every frame (`useFrame((state, delta) => { ... })`):
     - Extract `leftHandPos` and `rightHandPos` from `getState()`.
     - **Distance Check:** Calculate 3D euclidean distance between the two points: `d = Math.sqrt((Lx - Rx)**2 + (Ly - Ry)**2 + (Lz - Rz)**2)`.
     - **Collision Context:** Define an `INTERSECTION_THRESHOLD` (e.g., `1.5` units, depending on sphere size).
     - **Velocity Calculation:**
       - If `d < INTERSECTION_THRESHOLD`, the objects are "rubbing".
       - Calculate the distance the hands moved since last frame.
       - Calculate $v_{rel} = \frac{\Delta d}{\Delta t}$ (where $\Delta t$ is the `delta` parameter from `useFrame`).
       - Apply simple exponential smoothing to $v_{rel}$ to prevent lag spikes from causing infinite charge explosions. E.g., `smoothV = 0.8 * prevSmoothV + 0.2 * currentV`.
     - **Trigger Physics Engine:**
       - If rubbing and `smoothV > 0.01` (noise threshold):
       - Call `useSimulationStore.getState().applyFriction(smoothV, delta)`.
     - Finally, store current positions to the `useRef` for next frame comparison.

**Verification:**
- With the hands far apart, the HUD charges remain stationary.
- Bringing the hands closely together and making a "rubbing" motion causes $q_1$ and $q_2$ to rapidly diverge based on the selected materials.
- Stopping the rubbing instantly halts charge generation.
