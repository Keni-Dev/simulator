# Phase 4.1: Render Hidden Video & Initialize MediaPipe

**Phase:** 4 - Computer Vision (MediaPipe) Integration
**Step:** 4.1

**Context:**
This phase bridges the physical world (webcam) to the 3D simulation. It requires requesting camera permissions and continuously piping video frames into the `@mediapipe/hands` inference engine.

**Skills to Load:**
- `research-engineer` — For rigorous setup of the MediaPipe asynchronous loading and inference loop.

**Instructions:**

1. **Create `src/components/vision/HandTracker.tsx`:**
   - Scaffold a client component (`"use client"`).
   - This component should render *nothing* visually except maybe a hidden `<video>` element for capturing the stream.
   - Import `Hands` from `@mediapipe/hands` and `Camera` from `@mediapipe/camera_utils`.

2. **Implement Webcam Capture & Inference (useEffect):**
   - Create a `videoRef`.
   - On mount, instantiate the `Hands` model with standard options (`maxNumHands: 2`, `modelComplexity: 1`, `minDetectionConfidence: 0.5`, `minTrackingConfidence: 0.5`).
   - Define the `.onResults(results)` callback.
   - Attach the `Camera` utility to the `videoRef` and the `Hands.send({ image: video })` function. Start the camera.

3. **Process Inference Results (onResults Callback):**
   - Inside the callback, check if `results.multiHandLandmarks` and `results.multiHandedness` are present.
   - Iterate through the detected hands (up to 2).
   - For each hand, extract its handedness (`'Left'` or `'Right'`). *Note: Webcams are often mirrored. Map 'Left' to 'right' store state and vice-versa if it feels backward to the user.*
   - Determine the central coordinate of the hand. You can simply use `landmarks[9]` (the middle finger MCP joint) as a proxy for the palm center, or average `[0, 5, 9, 13, 17]`.
   - **Coordinate Mapping (CRITICAL):**
     - MediaPipe returns normalized coordinates (0.0 to 1.0) where `(0,0)` is top-left.
     - R3F canvas is standard cartesian where `(0,0)` is the center.
     - Map `x`: `(landmark.x - 0.5) * 10` (adjust `10` based on your camera frustum size).
     - Map `y`: `-(landmark.y - 0.5) * 10` (invert Y so moving hand up moves sphere up).
     - Define `z = 0` for MVP.
   - Push the computed `[X, Y, Z]` to the Zustand store via `updateHandPos('left', [X,Y,Z])` or `'right'`.

**Verification:**
- Accepting the browser camera prompt successfully turns on the webcam LED.
- Waving hands in front of the camera causes the 3D placeholder spheres in the simulator to mirror your movements on screen reliably.
