# Phase 3.1: Build the Simulation Canvas Wrapper

**Phase:** 3 - The Simulator Canvas & UI HUD
**Step:** 3.1

**Context:**
This is the heart of the application. The `/simulator` route contains a full-screen React Three Fiber `<Canvas>` underpinning the highly technical DOM-based HUD. This step sets up the empty 3D scene holding placeholder geometric representations of the hands.

**Skills to Load:**
- `3d-web-experience` — To bootstrap the R3F `<Canvas>` correctly behind a DOM overlay layer.
- `react-ui-patterns` — For efficient handling of full-viewport components.

**Instructions:**

1. **Create `src/components/scene/SimulationCanvas.tsx`:**
   - Scaffold a client component (`"use client"`).
   - Import `{ Canvas }` from `@react-three/fiber` and `{ Sphere }` from `@react-three/drei`.
   - Setup a `<Canvas>` that takes up `100vw` and `100vh`. Add basic lighting: `<ambientLight intensity={0.5} />` and `<directionalLight position={[10, 10, 5]} intensity={1} />`.
   - Connect to the store: `const { leftHandPos, rightHandPos } = useSimulationStore()`.
   - Render two placeholder `<Sphere args={[1, 32, 32]}>` meshes. Map the left sphere's `position` prop to `leftHandPos` and the right sphere's `position` to `rightHandPos`. Apply simple standard materials (e.g., `#00E5FF` for left, `#FF3D00` for right) just so they are visible.

2. **Create `src/app/simulator/page.tsx`:**
   - Default export a Client Component representing the simulation view.
   - Use a full viewport wrapper (`100vw`, `100vh`, `relative`, `overflow-hidden`).
   - Render `<SimulationCanvas />` absolutely positioned behind everything (`absolute inset-0 z-0`).

**Verification:**
- Navigating to `/simulator` shows a pitch black 3D canvas with two distinct, poorly lit spheres floating in space representing the default hand positions.
