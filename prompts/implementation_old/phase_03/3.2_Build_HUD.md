# Phase 3.2: Build the HUD Components

**Phase:** 3 - The Simulator Canvas & UI HUD
**Step:** 3.2

**Context:**
Implements the exact UI design requirements described in `02_main_simulator_hud.md`. This HUD sits on top of the R3F layer via `z-10` and reads directly from the physics Zustand store.

**Skills to Load:**
- `frontend-design` — For the strict SpaceX-style minimalist layout, neon typography, and backdrop blurs.
- `ui-ux-pro-max` — Handles exact variable setups for Tailwind.

**Instructions:**

1. **Create `src/components/hud/TopMetricsBar.tsx`:**
   - Client Component. Connects to `useSimulationStore()` to read `qNet`.
   - Fixed top, full width, 1px bottom border `#2A2A2D`, dark backdrop blur.
   - Left: `<span className="font-mono text-zinc-500 text-xs tracking-widest">CAMERA MATRIX: OK</span>`. Add a pulsing green status dot next to it.
   - Center: The core formula readout. `<span className="font-bold text-white text-lg">Σq (Net Charge) = {qNet.toFixed(2)} µC</span>`. *Critical detail:* To visualize conservation, `qNet` should effectively be `0.00` constantly. Render a faint glowing pill around this element to emphasize it.
   - Right: Render a minimal "EXIT" button returning to `/`.

2. **Create `src/components/hud/HandTelemetryPanel.tsx`:**
   - Client Component. Receives a prop `hand: 'left' | 'right'`.
   - Connects to `useSimulationStore()` to read `q1` (if left), `q2` (if right), and `materialId`.
   - Structural: A vertical panel positioned strictly on the left or right edges of the screen `w-64`, vertically centered.
   - Header: `font-inter text-[10px] tracking-widest text-zinc-500` reading `MATERIAL A (LEFT)` or `MATERIAL B (RIGHT)`.
   - Main Metric: The absolute charge. E.g., `q1.toFixed(2) µC`. The typography is `font-jetbrains text-5xl`.
     - *Color Logic:* If `q < 0`, text is `text-neon-cyan` with `[text-shadow:0_0_15px_rgba(0,229,255,0.5)]`. If `q > 0`, text is `text-neon-red` with identical red shadow logic. If `0.00`, text is white.
   - Interactive: A native HTML `<select>` styled minimally (dark background, white text) populating the options directly from `MATERIALS` defined in `triboelectric.ts`. `onChange` triggers the store's `setMaterial()` action.

3. **Create `src/components/hud/ActionActionBar.tsx`:**
   - Absolute bottom-center, hovering generic action panel.
   - Contains a solid, `border-zinc-700` button that says `[SPACE] Ground / Discharge`.
   - Attach a `useEffect` handling the global `'keydown'` listener targeting `Space`. Both clicking the button and pressing space trigger `useSimulationStore.getState().groundCharges()`.

4. **Integrate into `src/app/simulator/page.tsx`:**
   - Add `<TopMetricsBar />`, two instances of `<HandTelemetryPanel />` (left/right mapped), and `<ActionActionBar />` onto the layout underneath `SimulationCanvas` via `absolute z-10`.

**Verification:**
- The HUD matches the exact sci-fi typography specs mapped in the prompt.
- Changing materials updates the Zustand store correctly.
- Discharging wipes everything to `0.00`.
