# Phase 5.2: Directional Particle Flow (Electron Transfer VFX)

**Phase:** 5 - 3D Assets, VFX, and Polish
**Step:** 5.2

**Context:**
This is the "Wow Factor" of the simulator. When rubbing occurs, we physically show electrons (negative charge) transferring from one material to the other via a glowing particle stream.

**Skills to Load:**
- `3d-web-experience` — Critical for high-performance `InstancedMesh` usage. Do not use individual standard `<mesh>` components for particles as that will drop FPS instantly on an M1 Air.
- `web-performance-optimization` — Ensure 60fps operation.

**Instructions:**

1. **Create `src/components/scene/vfx/ElectronParticles.tsx`:**
   - Client Component utilizing `useFrame`.
   - We need to visualize thousands of tiny cyan glowing specks flying between the two hands.

2. **Implement Particle System (`InstancedMesh`):**
   - Import `<InstancedMesh>` from R3F. Allocate memory for ~1000 instances (`args={[geometry, material, 1000]}`).
   - Define a tiny `<SphereGeometry>` and a `<meshBasicMaterial color="#00E5FF" />`.

3. **Drive Particle Logic (`useFrame`):**
   - Maintain an array of active particles with `{ position, velocity, target, ttl }`.
   - Read the current $v_{rel}$ or $\Delta q$ from Zustand (or from the parent `SimulationCanvas`).
   - If $\Delta q$ is sufficient (materials are rubbing):
     - Determine the *donor* and *acceptor* material. Electrons *always* flow from the negatively indexed material (e.g., Teflon) *to* the positively indexed material (e.g., Glass).
     - Spawn new particles at the 3D coordinates of the donor material.
     - Set their `target` to the coordinates of the acceptor material.
   - For all active particles:
     - Mutate their `position` toward their `target` using `MathUtils.lerp()`.
     - Update the `InstancedMesh` matrix at their index (`dummy.position.set(...)`, `dummy.updateMatrix()`, `mesh.setMatrixAt(i, dummy.matrix)`).
     - Flag `mesh.instanceMatrix.needsUpdate = true`.
     - Decrease `ttl`. If `ttl < 0` or they reach the target, recycle the particle.

4. **Conductor Grounding Effect (Bonus Detail):**
   - If `groundCharges()` is triggered (or a conductor is selected), instantly spawn a cascade of particles moving rapidly *downward* (offscreen) to simulate dissipating into the human body/ground.

5. **Scene Composition (`SimulationCanvas.tsx`):**
   - Include `<PhysicalMaterial hand="left"/>` and `<PhysicalMaterial hand="right"/>`.
   - Add `<ElectronParticles />`.
   - Wrap everything in `<EffectComposer>` and add an intense `<Bloom luminanceThreshold={0.5} mipmapBlur intensity={1.5} />` to make the particles actually glow.

**Verification:**
- Rubbing "Glass" + "Teflon" clearly shows thousands of tiny cyan particles streaming from the Teflon to the Glass.
- Activating "Copper" visually grounds the charge immediately into the floor.
- Frames per second remains a stable 60.
