# Phase 0 — Step 0.3: Zustand Store Setup

## Context
The simulation requires high-performance state management that updates at ~60fps without causing React re-renders on every frame. Zustand is ideal because it allows the physics loop to write state imperatively (via `getState().setState()`) while React components subscribe only to the slices they need. This step creates the foundational store that every subsequent phase reads from and writes to.

- `react-patterns` — Selector patterns to prevent unnecessary re-renders

## Instructions

### Step 0.3.1: Create the Simulation Store
**File:** `src/stores/simulation-store.ts` — CREATE
**Purpose:** Central state for the physics simulation — materials, charges, collision status, and zoom state
**Details:**
- Create a Zustand store using `create` with the following state shape:
  - `leftMaterial`: Material (default to first material in MATERIALS array — e.g., Glass)
  - `rightMaterial`: Material (default to second material — e.g., Silk)
  - `q1`: number (charge on left object in μC, initial: 0)
  - `q2`: number (charge on right object in μC, initial: 0)
  - `isGrounding`: boolean (initial: false)
  - `isColliding`: boolean (initial: false)
  - `relativeVelocity`: number (initial: 0)
  - `isZooming`: boolean (initial: false)
  - `zoomLevel`: number (initial: 1.0 — multiplier for camera FOV)
  - `leftPosition`: `[number, number, number]` tuple (initial from SCENE_CONSTANTS.LEFT_SPAWN_POSITION)
  - `rightPosition`: `[number, number, number]` tuple (initial from SCENE_CONSTANTS.RIGHT_SPAWN_POSITION)
- Define actions (functions that update state):
  - `setLeftMaterial(material: Material)` — swaps the left material, resets q1 to 0
  - `setRightMaterial(material: Material)` — swaps the right material, resets q2 to 0
  - `updateCharges(deltaQ: number)` — adds deltaQ to q1, subtracts deltaQ from q2 (conservation: q1 + q2 always = 0 for insulators). Clamp both to ±MAX_CHARGE
  - `groundCharge(side: 'left' | 'right', amount: number)` — reduces charge toward 0 on the specified side by the given amount (for conductors)
  - `setCollisionState(isColliding: boolean, relativeVelocity: number)` — updates collision and velocity
  - `updateLeftPosition(position: [number, number, number])` — for hand-tracking position updates
  - `updateRightPosition(position: [number, number, number])` — for hand-tracking position updates
  - `setZooming(isZooming: boolean)` — toggles zoom state
  - `updateZoomLevel(delta: number)` — adjusts zoom level, clamped between 0.5 and 3.0
  - `resetSimulation()` — resets all charges to 0, positions to defaults, zoom to 1.0
- Use `immer` middleware if desired for immutable updates, but given the flat state shape, direct spreading is fine
- CRITICAL: Do NOT use `subscribeWithSelector` unless needed — keep subscriptions granular via selectors in components

### Step 0.3.2: Create the Hand Tracking Store
**File:** `src/stores/hand-tracking-store.ts` — CREATE
**Purpose:** Stores raw hand tracking data from MediaPipe — landmarks, gesture classification, palm positions, and velocities
**Details:**
- Create a Zustand store with state:
  - `isTracking`: boolean (initial: false — true once MediaPipe is initialized and detecting)
  - `isCameraReady`: boolean (initial: false — true once webcam permissions granted)
  - `leftHand`: HandState | null (initial: null — null means no left hand detected)
  - `rightHand`: HandState | null (initial: null)
  - `error`: string | null (initial: null — stores any MediaPipe/camera errors)
- Define actions:
  - `setCameraReady(ready: boolean)` — called when webcam stream is available
  - `setTracking(tracking: boolean)` — called when MediaPipe starts processing frames
  - `updateLeftHand(hand: HandState | null)` — updates left hand data every frame
  - `updateRightHand(hand: HandState | null)` — updates right hand data every frame
  - `setError(error: string | null)` — stores error messages for UI display
  - `reset()` — resets all tracking state
- This store will be updated at ~30fps from the MediaPipe callback. The key performance consideration: update the store directly without triggering unnecessary React renders. Components should use selectors to pick only the specific fields they need

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Store updates don't reflect in components | Selector not picking up changes | Ensure selector returns a new reference for objects, or use `shallow` equality |
| Entire app re-renders on every store update | Subscribing to entire store instead of specific fields | Use `useSimulationStore(state => state.q1)` not `useSimulationStore()` |
| "Cannot read property of undefined" on store | Store accessed before provider/initialization | Zustand doesn't need a provider — check if the module is imported correctly |
| Charges don't sum to zero | Bug in `updateCharges` — not applying equal and opposite | Verify: q1 += deltaQ, q2 -= deltaQ in the same action call |
