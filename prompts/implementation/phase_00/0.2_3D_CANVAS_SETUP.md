## Instructions

### Step 0.2.1: Create the Simulation Canvas Container
**File:** `src/features/simulation/components/SimulationCanvas.tsx` ŌĆö CREATE
**Purpose:** The top-level wrapper that mounts the R3F `<Canvas>` component with correct configuration
**Details:**
- Create a React component that renders the `@react-three/fiber` `<Canvas>` element
- Canvas props to set:
  - `camera`: position from `SCENE_CONSTANTS.CAMERA_DEFAULT_POSITION`, fov from `SCENE_CONSTANTS.CAMERA_DEFAULT_FOV`
  - `gl`: set `antialias: true`, `alpha: false` (opaque dark background), `powerPreference: 'high-performance'`
  - `dpr`: clamp to `[1, 2]` for retina displays without killing performance
  - `style`: full viewport width and height
- Inside the Canvas, render the `<Scene />` component (created in next step)
- Wrap the Canvas in a div that takes full viewport height with the dark background color (`#0f0f11`)
- The Canvas MUST have `'use client'` directive at the top of the file since it uses browser APIs

### Step 0.2.2: Create the Scene Component
**File:** `src/features/simulation/components/Scene.tsx` ŌĆö CREATE
**Purpose:** Defines the 3D scene contents ŌĆö lighting, environment, grid, and the two material mesh placeholders
**Details:**
- This is a component rendered INSIDE the `<Canvas>` (it's a Three.js scene child)
- Add lighting:
  - One `ambientLight` with intensity ~0.3 (dim ambient for the dark mood)
  - One `pointLight` positioned above and in front (y: 5, z: 5) with intensity ~1.0, white color
  - One subtle `pointLight` behind the objects for backlight rim effect (y: 2, z: -3), intensity ~0.5, slight blue tint
- Add an optional subtle `gridHelper` on the ground plane (size: 20, divisions: 20, faded color like `#1a1a2e`) to give spatial reference ŌĆö make it conditional on a debug prop
- Render two `<MaterialMesh />` components:
  - Left mesh at `SCENE_CONSTANTS.LEFT_SPAWN_POSITION`
  - Right mesh at `SCENE_CONSTANTS.RIGHT_SPAWN_POSITION`
  - Pass the default materials from `MATERIALS[0]` and `MATERIALS[1]` as props
- Add `@react-three/drei`'s `<OrbitControls>` temporarily for development camera control (will be replaced by hand-tracking zoom in Phase 1). Set `enablePan: false`, `enableZoom: true`, `enableRotate: true`

### Step 0.2.3: Create the MaterialMesh Component
**File:** `src/features/simulation/components/MaterialMesh.tsx` ŌĆö CREATE
**Purpose:** Renders a single 3D object representing a material. For now, a simple colored cube placeholder
**Details:**
- Accept props: `material` (Material type), `position` (Three.js Vector3 or [x, y, z] tuple), `charge` (number, default 0)
- Render a `<mesh>` with a `<boxGeometry>` (args: [1, 1, 1]) as placeholder ŌĆö these will be replaced with proper material-specific shapes in Phase 3
- Apply a `<meshStandardMaterial>` with:
  - `color` from the material's color property
  - `emissive` color that intensifies based on charge value (red for positive, blue for negative, no glow at 0)
  - `emissiveIntensity` proportional to `Math.abs(charge) / PHYSICS_CONSTANTS.MAX_CHARGE`
  - `metalness` and `roughness` set based on material type (conductors: metalness 0.8, roughness 0.2; insulators: metalness 0.1, roughness 0.7)
- Use `useRef` to get a reference to the mesh for future physics collision detection
- Add the mesh to a group so position and rotation can be controlled independently

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Canvas renders white/blank | Background color not set or Canvas `alpha` is true | Set `alpha: false` on Canvas `gl` prop and parent div background to `#0f0f11` |
| "Cannot use import statement outside a module" for Three.js | Server-side rendering issue | Add `'use client'` directive and dynamically import the Canvas with `next/dynamic` using `ssr: false` |
| OrbitControls don't respond | Event listeners not attached to Canvas | Ensure OrbitControls is rendered INSIDE the `<Canvas>`, not outside |
| Performance warning about device pixel ratio | DPR too high on retina screens | Clamp `dpr` prop on Canvas to `[1, 2]` |
| Cubes appear but are completely black | No lights in scene | Verify `ambientLight` and `pointLight` are children of the scene |
