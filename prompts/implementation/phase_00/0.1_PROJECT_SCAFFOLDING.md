# Phase 0 — Step 0.1: Project Scaffolding

## Context
This is the very first step of the entire build. It creates the Next.js project with TypeScript strict mode, installs all core dependencies, and establishes the directory structure that every subsequent phase builds upon. Nothing else can happen until this step is complete.

## Instructions

### Step 0.1.1: Initialize Next.js Project
**Action:** CREATE project using `npx create-next-app@latest`
**Details:**
- Run `npx create-next-app@latest ./ --typescript --tailwind --eslint --app --src-dir --import-alias "@/*"` in the project root directory
- Select **No** for Turbopack if prompted (stability preference)
- This generates the foundational Next.js App Router project with TypeScript and Tailwind CSS preconfigured

### Step 0.1.2: Install Core Dependencies
**Action:** INSTALL packages
**Details:**
- Install 3D rendering: `three @react-three/fiber @react-three/drei` — Three.js with React bindings and helper components
- Install state management: `zustand` — High-performance state management for the continuous physics loop
- Install type definitions: `@types/three` — TypeScript definitions for Three.js
- Install MediaPipe: `@mediapipe/hands @mediapipe/camera_utils @mediapipe/drawing_utils` — Client-side hand tracking (verify latest versions, MediaPipe may now use `@mediapipe/tasks-vision` as the consolidated package)
- Install fonts: `@fontsource/jetbrains-mono @fontsource/inter` — Monospace for scientific readouts, sans-serif for labels

### Step 0.1.3: Configure TypeScript Strict Mode
**File:** `tsconfig.json` — MODIFY
**Purpose:** Enforce strict typing to prevent math and state bugs in the physics engine
**Details:**
- Set `"strict": true` (should already be true from create-next-app)
- Add `"noUncheckedIndexedAccess": true` — catches potential undefined array access in landmark arrays
- Add `"exactOptionalPropertyTypes": true` — stricter optional property handling
- Ensure `"paths"` includes `"@/*": ["./src/*"]` for clean imports

### Step 0.1.4: Create Feature-Based Directory Structure
**Action:** CREATE directories
**Details:**
- Create `src/features/simulation/` — will hold the main 3D canvas, scene setup, and material meshes
- Create `src/features/hand-tracking/` — will hold MediaPipe integration, gesture detection, and hand landmark processing
- Create `src/features/physics/` — will hold the triboelectric math engine, collision detection, and charge state logic
- Create `src/features/hud/` — will hold the overlay HUD components showing charge values and controls
- Create `src/components/` — shared UI components (buttons, overlays, warnings)
- Create `src/stores/` — Zustand store definitions
- Create `src/types/` — shared TypeScript type definitions and interfaces
- Create `src/constants/` — material data, triboelectric indices, friction coefficients
- Create `src/utils/` — pure utility functions (math helpers, vector operations)
- Create `src/config/` — typed environment configuration
- Create `public/models/` — will hold 3D model files or textures if needed

### Step 0.1.5: Create Type Definitions Foundation
**File:** `src/types/index.ts` — CREATE
**Purpose:** Define all shared TypeScript types used across the application
**Details:**
- Define a `MaterialType` enum with values `'insulator'` and `'conductor'`
- Define a `Material` interface with fields: `id` (string), `name` (string), `type` (MaterialType), `triboIndex` (number), `modelPath` (string — path to 3D model or empty for basic geometry), `color` (string — hex color for the mesh)
- Define a `SimulationState` interface with fields: `leftMaterial` (Material), `rightMaterial` (Material), `q1` (number — charge in μC on left object), `q2` (number — charge in μC on right object), `isGrounding` (boolean — true when a conductor is touching the hand), `isZooming` (boolean — true when zoom gesture active), `leftPosition` (a 3D vector type for left object position), `rightPosition` (a 3D vector type for right object position), `relativeVelocity` (number — calculated from hand movements), `isColliding` (boolean — true when bounding boxes of left and right objects intersect)
- Define a `HandGesture` enum: `'none'`, `'grab'`, `'zoom'`
- Define a `HandState` interface with fields: `detected` (boolean), `landmarks` (array of 21 3D points), `gesture` (HandGesture), `position` (3D vector — palm center), `velocity` (3D vector — palm movement velocity)
- Export all types

### Step 0.1.6: Create Constants Foundation
**File:** `src/constants/index.ts` — CREATE
**Purpose:** Centralize all magic numbers and configuration values
**Details:**
- Define `MATERIALS` — an array of `Material` objects. Include at minimum:
  - Glass (insulator, triboIndex: +25)
  - Silk (insulator, triboIndex: +5)
  - Rubber (insulator, triboIndex: -70)
  - Fur (insulator, triboIndex: +15)
  - Copper (conductor, triboIndex: 0)
- Define `FRICTION_COEFFICIENTS` — a 2D lookup matrix mapping material ID pairs to their friction coefficient `k`. Higher k = faster charge transfer. Example: Glass-Silk has a high k (~0.8), Rubber-Fur has a medium k (~0.5)
- Define `PHYSICS_CONSTANTS` — object containing: `MAX_CHARGE` (maximum absolute μC a material can hold, e.g., 100), `GROUNDING_RATE` (how fast conductors discharge, e.g., 50 μC/s), `CHARGE_PRECISION` (decimal places for display, e.g., 2)
- Define `SCENE_CONSTANTS` — object containing: `LEFT_SPAWN_POSITION` (x: -2, y: 0, z: 0), `RIGHT_SPAWN_POSITION` (x: 2, y: 0, z: 0), `CAMERA_DEFAULT_POSITION`, `CAMERA_DEFAULT_FOV`
- Define `HAND_TRACKING_CONSTANTS` — object containing: `PINCH_THRESHOLD` (distance in normalized coordinates to consider thumb+index touching, e.g., 0.05), `FINGER_OPEN_THRESHOLD` (angle threshold for considering a finger "open"), `FINGER_CLOSED_THRESHOLD` (angle threshold for "closed")
- All values as `as const` for literal types

### Step 0.1.7: Create Environment Config
**File:** `src/config/env.ts` — CREATE
**Purpose:** Typed access to environment variables (even though minimal for a client-side app)
**Details:**
- Create a typed `config` object that reads from `process.env`
- Include: `NEXT_PUBLIC_APP_NAME` (default: "TriboSim"), `NEXT_PUBLIC_DEBUG_MODE` (boolean, default: false — enables console logging of physics values)
- Export the config object

## Rollback Plan
If this step breaks something:
- Delete the entire project directory and re-run `create-next-app`
- Or revert to the initial git commit: `git checkout -- .`

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| `create-next-app` fails | Node.js version too old | Upgrade to Node.js 20+ |
| TypeScript errors on Three.js types | Missing `@types/three` | Run `npm install @types/three` |
| Tailwind not working | PostCSS config missing | Verify `tailwind.config.ts` and `postcss.config.mjs` exist |
| Import alias `@/` not resolving | `tsconfig.json` paths misconfigured | Ensure `"@/*": ["./src/*"]` is in `compilerOptions.paths` |
