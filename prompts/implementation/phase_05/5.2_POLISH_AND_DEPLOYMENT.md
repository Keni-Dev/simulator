## Instructions

### Step 5.2.1: Create Instruction Overlay
**File:** `src/components/InstructionOverlay.tsx` — CREATE
**Purpose:** First-time user instructions explaining how to use the simulator
**Details:**
- On first load (or when a "Help" button is clicked), display a semi-transparent overlay with step-by-step instructions:
  1. "Allow camera access to enable hand tracking"
  2. "Pinch your thumb and index finger (keep other 3 fingers open) to GRAB objects"
  3. "Move your hands to rub the two materials together"
  4. "Watch the HUD for charge transfer and conservation proof"
  5. "Pinch with all fingers closed to ZOOM the camera"
  6. "Use the ⚙️ panel to swap materials"
- Include small illustrations or icons for each gesture (simple SVG hand diagrams)
- "Got it!" button dismisses the overlay
- Store dismissal state in `localStorage` so it doesn't show on reload
- Can be re-opened via a "?" help button in the HUD corner
- Use Entry/exit animations (fade in, slide up)
- Styling: dark overlay with bright white text, consistent with the HUD aesthetic

### Step 5.2.3: Fine-Tune Friction Coefficients
**File:** `src/constants/index.ts` — MODIFY
**Purpose:** Adjust the `k` values in the `FRICTION_COEFFICIENTS` matrix for the best "fun factor"
**Details:**
- The goal: charge transfer should feel noticeable within 2-3 seconds of moderate rubbing, and reach max charge within 10-15 seconds of vigorous rubbing
- Tuning guidelines:
  - Glass + Silk: high k (~0.8-1.0) — these are the classic triboelectric pair, should produce fast, dramatic charge transfer
  - Rubber + Fur: medium k (~0.5-0.7) — noticeable but slower than Glass+Silk
  - Any material with itself: k = 0 — no charge transfer
  - Conductor pairs: k doesn't matter as much since grounding dominates
- Also tune `TRIBO_NORMALIZATION_FACTOR` — this scales the raw equation output to μC range
- Also tune `GROUNDING_RATE` for conductors — should discharge to 0 within ~1-2 seconds of contact (fast, dramatic spark)
- Also tune `MAX_CHARGE` — 100 μC is a reasonable cap; reduce if charges grow too quickly
- These values are best tuned by running the app and observing the experience. Adjust iteratively

### Step 5.2.4: Handle Edge Cases
**Action:** Review and handle across multiple files
**Details:**
- **Hands leaving camera frame:**
  - When a hand disappears during a grab, the object should stay in its last position (not snap back)
  - Gesture state resets to `'none'` when hand is lost
- **Both hands on same object:**
  - Only the first hand to grab an object controls it; the other hand can grab the other object
  - Or: if both hands are near the same object, the closer hand takes priority
- **Rapid material switching:**
  - Debounce material selection changes to prevent rapid-fire mesh swaps (200ms debounce)
  - Reset VFX cleanly when swapping — no orphaned particles from the previous material
- **Browser tab backgrounded:**
  - When the tab is hidden (`document.visibilityState === 'hidden'`), pause MediaPipe and the physics loop to save CPU
  - Resume when tab is visible again
- **Window resize:**
  - Handle canvas resize properly (R3F handles this automatically, but verify the HUD repositions correctly)
- **Very low light / hand occlusion:**
  - If MediaPipe detection confidence drops below threshold, show a hint: "Move to better lighting" in the status bar
- **Long-running session:**
  - After 30+ minutes, verify no memory leaks (Three.js textures, particle arrays, MediaPipe buffers)

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Vercel build fails | MediaPipe WASM files or server-side rendering issues | Ensure all browser-only code uses `'use client'` and dynamic imports with `ssr: false` |
| Camera doesn't work on production | HTTPS required for webcam access | Vercel serves HTTPS by default — this should work. If using custom domain, ensure SSL is configured |
| Particles look different in production | Different GPU/driver rendering | Test on target hardware (M1 MacBook). Adjust particle size/opacity if needed |
| Page load is slow (>5s) | Large bundle or unoptimized assets | Run `npm run build` and check bundle size. Lazy-load MediaPipe and Three.js |
| localStorage instruction dismissal doesn't persist | Incognito mode or storage full | Fall back to showing instructions if localStorage is unavailable |
