# Phase 4 — Step 4.2: Material Selector UI

## Context
The Material Selector allows users to swap the left and right materials to test different triboelectric combinations (e.g., Glass vs Silk, Rubber vs Fur, any material vs Copper conductor). It completes the interactive loop: select materials → grab → rub → observe charge transfer → try another combination. This is the main user control panel.

## Skills to Load
- `ui-ux-pro-max` — Interactive component design, hover states, selection states
- `frontend-design` — Premium UI for the selector panel matching the sci-fi aesthetic

## Prerequisites
- Step 4.1 complete: HUD overlay rendering on top of the canvas
- Constants defined: `MATERIALS` array with all 5+ materials
- Simulation store has `setLeftMaterial()` and `setRightMaterial()` actions

## Instructions

### Step 4.2.1: Create Material Card Component
**File:** `src/features/hud/components/MaterialCard.tsx` — CREATE
**Purpose:** A single selectable material option in the selector panel
**Details:**
- Accept props: `material` (Material), `isSelected` (boolean), `onSelect` (callback)
- Display:
  - Material name (e.g., "Glass")
  - Material type badge: "Insulator" (blue badge) or "Conductor" (gold badge)
  - A small colored circle/swatch showing the material's color
  - Triboelectric index: "T: +25"
- Selected state:
  - Bright border (white or the primary accent color)
  - Slightly elevated (box-shadow)
  - Background slightly lighter than unselected
- Unselected hover state:
  - Subtle border highlight
  - Scale up slightly (transform: scale(1.02))
  - Cursor: pointer
- Styling:
  - Dark card background (`rgba(255, 255, 255, 0.05)`)
  - Rounded corners (8px)
  - Font: Inter for label, JetBrains Mono for the triboIndex number
  - Compact size — the card should be small enough that 5+ materials fit without scrolling
- Must have `pointer-events: auto` since it needs click interactions
- Include `role="button"` and keyboard accessibility (focusable, Enter/Space to select)

### Step 4.2.2: Create Material Selector Component
**File:** `src/features/hud/components/MaterialSelector.tsx` — CREATE
**Purpose:** A panel showing all available materials, letting the user assign any material to the left or right spawn position
**Details:**
- Layout: a slide-out panel on the left or right side of the screen, or inline in the HUD
- Two selection columns:
  - "Left Object" — shows all materials; clicking one calls `setLeftMaterial(material)` and resets charges
  - "Right Object" — same, calls `setRightMaterial(material)`
- Display the current selection highlighted in each column
- When a material is selected:
  1. Update the simulation store (`setLeftMaterial` or `setRightMaterial`)
  2. Reset both charges to 0 (new experiment)
  3. The 3D scene automatically updates (mesh shape/material changes, VFX reset)
- Add a "Swap ↔" button between the columns to quickly swap left and right materials
- Show a tooltip or info note: "Conductor materials (e.g., Copper) will ground charge through the human body"
- Panel can be toggled open/closed with a ⚙️ or 🧪 icon button in the corner — saves screen space
- Transition: slide in from the side with a smooth CSS transition (300ms ease-out)
- Panel background: same glassmorphism as the HUD bars

### Step 4.2.3: Create Reset Button
**File:** `src/features/hud/components/ResetButton.tsx` — CREATE
**Purpose:** Resets the simulation — clears all charges, resets object positions, and clears VFX
**Details:**
- A button labeled "Reset Experiment" or "🔄 Reset"
- On click: calls `useSimulationStore.getState().resetSimulation()`
- Positioned in the HUD — either in the status bar or near the material selector
- Styling: subtle outline button, not too prominent. Hover: slight glow, scale up
- Add a keyboard shortcut: pressing "R" key triggers reset (add a `useEffect` with `keydown` listener)
- Include an accessible label and tooltip

### Step 4.2.4: Integrate Selector into HUD

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Clicking material doesn't change the 3D mesh | Store update not triggering MaterialMesh re-render | Verify MaterialMesh subscribes to `leftMaterial` / `rightMaterial` from store |
| Selector panel overlaps charge displays | z-index or positioning conflict | Use proper stacking: selector panel should be below the top bar but above the canvas |
| Material card clicks aren't registering | `pointer-events: none` on parent | Add `pointer-events: auto` on the selector panel and individual cards |
| Reset doesn't clear VFX | Particle system not responding to charge reset | ElectronParticles and ChargeAura should read charge from store — when 0, VFX should stop |
