# Phase 4 — Step 4.1: Scientific HUD Overlay

## Context
The HUD (Heads-Up Display) is the scientific proof panel of the simulator. It displays real-time charge values ($q_1$, $q_2$), the net charge ($q_{net}$), and material names — overlaid on top of the 3D canvas. The HUD must prove conservation of charge visually: the student watches $q_{net}$ stay at $0\mu C$ as individual charges change. This is a React DOM overlay (not part of the Three.js scene) for sharp text rendering.

## Skills to Load
- `ui-ux-pro-max` — Design system for the sci-fi HUD aesthetic, monospace typography for scientific displays, contrast against dark backgrounds
- `frontend-design` — Premium overlay UI construction, glassmorphism, and subtle animation for dynamic numbers
- `react-patterns` — Efficient store subscriptions for frequently updating numeric displays

## Prerequisites
- Phase 3 complete: 3D scene with VFX fully functional
- Simulation store provides: `q1`, `q2`, `leftMaterial`, `rightMaterial`, `isColliding`, `relativeVelocity`
- Fonts loaded: JetBrains Mono for numbers, Inter for labels

## Instructions

### Step 4.1.1: Create HUD Stylesheet
**File:** `src/features/hud/styles/hud.module.css` — CREATE
**Purpose:** All HUD-specific styles using CSS Modules for scoping
**Details:**
- The HUD uses `position: fixed` to overlay the 3D canvas, with `pointer-events: none` so mouse/touch events pass through to the canvas beneath
- Interactive elements (buttons in the material selector) selectively re-enable `pointer-events: auto`
- Top bar styling:
  - Full width, fixed to top
  - Semi-transparent dark background (`rgba(15, 15, 17, 0.75)`) with `backdrop-filter: blur(12px)` for glassmorphism
  - Padding: 12px 24px
  - Border-bottom: 1px solid `rgba(255, 255, 255, 0.08)` — subtle separator
- Charge number styling:
  - Font: JetBrains Mono (monospace)
  - Size: 24px for charge values, 36px for net charge
  - Color: white by default, animate to red when positive, blue when negative
  - Use tabular-nums font feature for fixed-width digits (numbers don't shift as values change)
- Label styling:
  - Font: Inter (sans-serif)
  - Size: 12px, uppercase, letter-spacing: 0.1em
  - Color: `rgba(255, 255, 255, 0.5)` — muted labels
- Net charge zero indicator:
  - When `q_net = 0`: display a subtle green glow or checkmark icon
  - Text color: tech green (`#00ff00`)

### Step 4.1.2: Create Charge Display Component
**File:** `src/features/hud/components/ChargeDisplay.tsx` — CREATE
**Purpose:** Displays a single charge value with label, sign indicator, and color coding
**Details:**
- Accept props: `label` (string, e.g., "Material A"), `charge` (number), `materialName` (string)
- Display format: `±XX.XX μC` — always show 2 decimal places, prepend + or - sign
- Color animation:
  - charge = 0: white text
  - charge > 0: smoothly transitions to neon red/orange
  - charge < 0: smoothly transitions to neon cyan/blue
  - Use CSS transitions on the `color` property for smooth changes
- Add a subtle glow effect: `text-shadow` in the charge's color with blur radius 8px
- Show the material name below the charge value in the muted label style
- Use `useSimulationStore(state => state.q1)` (or q2) — fine-grained selector to prevent re-renders from other store changes
- Format numbers with `toFixed(PHYSICS_CONSTANTS.CHARGE_PRECISION)` from constants

### Step 4.1.3: Create Net Charge Indicator Component
**File:** `src/features/hud/components/NetChargeIndicator.tsx` — CREATE
**Purpose:** Displays $q_{net} = q_1 + q_2$ prominently, proving conservation of charge
**Details:**
- Compute `qNet = q1 + q2` from the store
- Display: `q_net = 0.00 μC` (or the actual value if floating-point drift causes deviation)
- When `qNet === 0` (within tolerance):
  - Display in green (`#00ff00`) with text "✓ Conservation Verified"
  - Subtle green glow effect
- When `qNet ≠ 0` (shouldn't happen, but safety):
  - Display in yellow/warning with text "⚠ Conservation Deviation Detected"
- Add the equation label: "$q_1 + q_2 = q_{net}$" rendered as: "q₁ + q₂ = qₙₑₜ" using subscript Unicode characters
- Position: center of the HUD top bar, between the two charge displays
- Font: JetBrains Mono, larger size (28-36px) to emphasize the key physics proof

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| HUD text appears behind the 3D canvas | z-index too low | Increase z-index on the overlay container. R3F Canvas creates its own stacking context |
| Numbers jitter/shift width as values change | Proportional font or no tabular-nums | Use JetBrains Mono (monospace) AND add `font-variant-numeric: tabular-nums` |
| Backdrop-filter blur not working | Browser support or GPU acceleration | Works in all modern browsers. Check if `backdrop-filter` has the `-webkit-` prefix |
| HUD blocks canvas interaction | `pointer-events: none` not set | Apply `pointer-events: none` to the overlay container |
| Charge updates cause entire HUD to re-render | Subscribing to full store instead of granular selectors | Each component must use its own selector: `useSimulationStore(s => s.q1)`, not `useSimulationStore()` |
