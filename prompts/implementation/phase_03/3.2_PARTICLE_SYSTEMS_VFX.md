# Phase 3 — Step 3.2: Particle Systems & VFX

## Context
This step creates the visual centerpiece of the simulator: particle effects that represent electron transfer between materials. When objects collide and charge transfers, directional particles flow from the electron-donating material to the electron-receiving material. Conductors produce a dramatic "spark" grounding effect. These VFX make the abstract physics concept tangible and visually prove charge conservation.

## Skills to Load
- `3d-web-experience` — Three.js particle systems (Points, BufferGeometry, InstancedMesh), shader materials for custom effects, performance optimization for particle counts
- `frontend-design` — Visual polish for the particle effects, ensuring they look premium on the dark background

## Prerequisites
- Step 3.1 complete: Material-specific meshes rendering with charge-based glow
- Physics engine running: `deltaQ` values calculated, collision state available
- Scene has dark background that makes glowing particles pop

## Instructions

### Step 3.2.1: Create Electron Transfer Particle System
**File:** `src/features/simulation/components/ElectronParticles.tsx` — CREATE
**Purpose:** Renders a stream of glowing particles flowing from the electron-donating material to the electron-receiving material during collision
**Details:**
- Create a particle system using Three.js `Points` with `BufferGeometry`:
  - Particle count: 50-100 active particles max (performance budget)
  - Each particle has: position (xyz), velocity (xyz), lifetime (seconds remaining), alpha (opacity)
- Particle behavior:
  1. **Spawn:** When `isColliding && relativeVelocity > 0`, spawn particles at the collision midpoint (average of left and right object positions)
  2. **Direction:** Particles flow FROM the material that LOSES electrons TO the material that GAINS electrons
     - If `deltaQ > 0`: flow from left (which becomes positive) to right (which becomes negative)
     - If `deltaQ < 0`: flow from right to left
     - Determine direction based on the triboelectric indices: electrons flow from higher triboIndex to lower triboIndex
  3. **Speed:** Particle velocity proportional to `relativeVelocity` — faster rubbing = faster-moving particles
  4. **Spawn rate:** Proportional to `Math.abs(deltaQ)` — more charge transfer = more particles
  5. **Lifetime:** Each particle lives for 0.5-1.5 seconds (randomized)
  6. **Fade:** Alpha decreases linearly over lifetime (full opacity at spawn, 0 at death)
- Visual appearance:
  - Each particle is a small glowing circle (use a `PointsMaterial` with `map` set to a soft radial gradient texture, or use `AdditiveBlending` for glow)
  - Color: Cyan/electric blue (`#00ccff`) — representing electrons
  - Size: 3-6 pixels (randomized)
  - Blending: `THREE.AdditiveBlending` for a glowing, luminous look
  - No depth testing (`depthWrite: false`) so particles glow through/above meshes
- Update loop (inside `useFrame`):
  - Move each particle along its velocity vector: `position += velocity * delta`
  - Decrease lifetime
  - Remove dead particles (lifetime <= 0)
  - Spawn new particles if collision is active
- When collision ends (`isColliding === false`):
  - Stop spawning new particles
  - Let existing particles fade out naturally
- PERFORMANCE: Use a particle pool pattern — pre-allocate positions in a Float32Array BufferGeometry and reuse particles from the pool instead of creating/destroying objects

### Step 3.2.2: Create Grounding Spark Effect
**File:** `src/features/simulation/components/GroundingSpark.tsx` — CREATE
**Purpose:** When a conductor material is charging (touching an insulator while the hand holds it), display a dramatic electrical spark/discharge effect on the conductor
**Details:**
- This effect triggers when: `isGrounding === true` AND one of the materials has `type === 'conductor'`
- Visual effect — bright white-yellow spark lines radiating from the conductor:
  - Use `Line` from `@react-three/drei` or custom `BufferGeometry` with `LineSegments`
  - Generate 3-5 jagged line segments starting from the conductor mesh surface and extending outward in random directions
  - Each line has 4-6 points creating a zigzag pattern (like a lightning bolt)
  - Lines appear briefly (0.1-0.3 seconds), then new ones spawn in different directions
  - Color: White (`#ffffff`) to bright yellow (`#ffff00`)
  - Use `AdditiveBlending` for glow
- Spark frequency: 2-4 sparks per second while grounding is active
- Add a subtle flash effect: briefly increase the conductor's `emissiveIntensity` when a spark fires
- When grounding stops, sparks fade out

### Step 3.2.3: Create Charge Aura Effect
**File:** `src/features/simulation/components/ChargeAura.tsx` — CREATE
**Purpose:** A persistent glow/aura around each material object that visualizes its current charge state — red for positive, blue for negative, no aura when neutral
**Details:**
- Create a slightly larger, transparent mesh that surrounds the material object (scaled to ~1.2x the object size)
- Use `MeshBasicMaterial` with:
  - `transparent: true`
  - `opacity`: scales from 0 (neutral) to 0.3 (max charge)
  - `color`: interpolate from neutral (transparent) to red (`#ff4444` positive) or blue (`#00ccff` negative) based on charge sign and magnitude
  - `side: THREE.BackSide` to render only the inside-facing surface, creating a halo effect
  - `blending: THREE.AdditiveBlending`
- The aura should pulse subtly: oscillate the opacity by ±10% at ~2Hz using `Math.sin(time * 4)`. This creates a "living energy" feel
- Scale the aura size based on charge magnitude: near-neutral = tight aura, high charge = larger expanded aura
- Accept `charge: number` as a prop (from store) and `position` matching the parent mesh

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Particles don't appear | Not spawning because `isColliding` is false | Debug collision detection from Phase 2. Also check if particles spawn at correct position |
| Particles look like flat white squares | Missing point texture or additive blending | Set `PointsMaterial` with `map` (radial gradient), `blending: AdditiveBlending`, `transparent: true` |
| Massive frame drop when particles appear | Too many particles or no pool pattern | Cap particle count at 50-100. Use BufferGeometry with pre-allocated positions, don't create new objects per particle |
| Aura clips through the material mesh | Aura mesh too close to material mesh | Scale aura to 1.3-1.5x instead of 1.2x. Use `renderOrder` to control drawing order |
| Sparks invisible on dark background | Color too dim | Use white/yellow with high emissive. Ensure `AdditiveBlending` is set |
