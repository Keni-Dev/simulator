# Phase 3 — Step 3.1: Material-Specific 3D Models & Textures

## Context
The placeholder cubes from Phase 0 are now replaced with visually distinct material representations. Each material gets a unique 3D shape and PBR (Physically Based Rendering) texture to make it instantly recognizable. This step focuses on visual identity — Glass looks like glass, Copper looks metallic, Silk looks soft. The recommendation from the spec is to use basic Three.js geometries with high-quality material settings rather than custom 3D model files, keeping the project lightweight for a school MVP.

- `3d-web-experience` — Three.js PBR materials, geometry types, and environmental lighting for realistic rendering
- `ui-ux-pro-max` — Color palette selection for material visualization that works on the dark background
- `frontend-design` — Ensuring the visual design is distinctive and premium despite using basic geometries

## Prerequisites

- MaterialMesh component exists and reads from the simulation store

## Instructions

### Step 3.1.1: Define Material Visual Configurations
**File:** `src/features/simulation/components/material-configs.ts` — CREATE
**Purpose:** Maps each material to its Three.js geometry type, material properties, and visual characteristics
**Details:**
- Create a `MaterialVisualConfig` type with fields:
  - `geometryType`: `'box'` | `'cylinder'` | `'sphere'` | `'torus'` | `'plane'` — which Three.js geometry to use
  - `geometryArgs`: number array for the geometry constructor (e.g., `[0.5, 1, 32]` for a cylinder radius, height, segments)
  - `materialProps`: object with fields: `color` (hex), `metalness` (0-1), `roughness` (0-1), `transparent` (boolean), `opacity` (0-1), `clearcoat` (0-1, for `MeshPhysicalMaterial`), `transmission` (0-1, for glass-like transparency)
  - `scale`: [x, y, z] tuple for the mesh scale
- Create the config map for each material:
  - **Glass** → Cylinder (like a glass rod). Geometry: CylinderGeometry(0.3, 0.3, 1.5, 32). Material: MeshPhysicalMaterial with `transmission: 0.7`, `roughness: 0.1`, `metalness: 0`, `clearcoat: 1.0`, slight blue-white tint. Looks like transparent glass
  - **Silk** → Plane/thin box (like a fabric swatch). Geometry: BoxGeometry(1.2, 0.8, 0.05). Material: MeshStandardMaterial with `roughness: 0.9`, `metalness: 0`, warm ivory/cream color. Looks like fabric
  - **Rubber** → Flattened sphere. Geometry: SphereGeometry(0.5, 32, 32). Material: MeshStandardMaterial with `roughness: 1.0`, `metalness: 0`, dark gray/black color. Looks like matte rubber
  - **Fur** → Small box with fuzzy appearance. Geometry: BoxGeometry(0.8, 0.6, 0.3). Material: MeshStandardMaterial with `roughness: 0.95`, `metalness: 0`, warm brown color. Could add subtle fuzz via (displacementMap or normal noise in future)
  - **Copper** → Cylinder/rod that looks metallic. Geometry: CylinderGeometry(0.25, 0.25, 1.2, 32). Material: MeshStandardMaterial with `metalness: 0.95`, `roughness: 0.3`, copper color `#b87333`. Clearly metallic

### Step 3.1.2: Update MaterialMesh Component
**File:** `src/features/simulation/components/MaterialMesh.tsx` — MODIFY (significant rewrite)
**Purpose:** Render the correct geometry and material for each material type, with dynamic charge-based visual effects
**Details:**
- Accept `side` prop: `'left'` | `'right'` to determine which store data to read
- Look up the `MaterialVisualConfig` for the current material
- Dynamically render the correct geometry based on `geometryType`:
  - Use a switch/map to select `<boxGeometry>`, `<cylinderGeometry>`, `<sphereGeometry>`, etc.
- Apply the configured material with PBR properties
- Add charge-based visual effects:
  - When charge > 0: mix the base color with red/orange (`--color-positive`), intensity proportional to charge magnitude
  - When charge < 0: mix the base color with blue/cyan (`--color-negative`)
  - Use `emissive` and `emissiveIntensity` for the glow effect
  - The glow should be subtle at low charges and pronounced at high charges
- Add a subtle idle animation: very slow rotation on the Y-axis (like the object is floating). Speed: ~0.1 radians per second. Use `useFrame` with `mesh.rotation.y += 0.001`
- Add a "grabbed" visual indication: when the corresponding hand gesture is `'grab'`, slightly scale up the mesh (1.05x) and increase brightness slightly — provides tactile feedback
- Use `useRef<THREE.Mesh>` to expose the mesh ref for collision detection

### Step 3.1.3: Add Environment Map for Reflections
**File:** `src/features/simulation/components/Scene.tsx` — MODIFY
**Purpose:** Add environment mapping for realistic PBR reflections, especially important for Glass and Copper materials
**Details:**
- Use `@react-three/drei`'s `<Environment>` component with `preset="night"` or `"city"` (dark environment for the sci-fi look)
- Alternative: use `<Environment background={false}>` with a custom HDRI or a simple procedural environment
- The environment map provides reflections on metallic/glass surfaces, making them look physically realistic without any textures
- Keep the background black (`#0f0f11`) — the environment only adds reflections, not background scenery

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Glass material looks completely opaque | `transmission` not working with MeshStandardMaterial | Switch to `MeshPhysicalMaterial` — only Physical material supports `transmission` |
| Copper doesn't look metallic | No environment map for reflections | Add `<Environment>` from drei — metal surfaces need something to reflect |
| Charge glow not visible | `emissiveIntensity` too low or emissive color same as base color | Increase intensity. Set `emissive` to a pure red/blue, separate from the base `color` |
| Transparent glass causes artifacts | Alpha sorting issues in Three.js | Set `depthWrite: false` on transparent materials or render transparent objects last |
