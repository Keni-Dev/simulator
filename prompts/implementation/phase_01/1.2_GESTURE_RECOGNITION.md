# Phase 1 — Step 1.2: Gesture Recognition System

## Context
With raw hand landmarks flowing into the store (Step 1.1), this step builds the gesture recognition layer. The app must distinguish between two specific hand gestures: **Grab** (thumb + index pinching, other 3 fingers open) for moving objects, and **Zoom** (thumb + index pinching, other 3 fingers closed) for camera zoom. This classification determines the interaction mode and is the foundation for all user input.

## Skills to Load
- `3d-web-experience` — Understanding how hand gesture data maps to 3D scene interactions
- `research-engineer` — Ensure the finger state detection math (angles, distances) is geometrically rigorous

## Prerequisites
- Step 1.1 complete: MediaPipe running, 21 landmarks per hand flowing into the hand-tracking store
- Hand landmark indices understood: thumb tip (4), index tip (8), middle tip (12), ring tip (16), pinky tip (20), and their corresponding MCP joints (5, 9, 13, 17)

## Instructions

### Step 1.2.1: Create Landmark Helper Utilities
**File:** `src/features/hand-tracking/utils/landmark-helpers.ts` — CREATE
**Purpose:** Pure utility functions for calculating distances, angles, and finger states from raw landmarks
**Details:**
- Create function `distance3D(pointA: {x, y, z}, pointB: {x, y, z}): number` — Euclidean distance between two 3D points
- Create function `isPinching(landmarks: Landmark[], threshold: number): boolean` — returns true if the distance between thumb tip (index 4) and index finger tip (index 8) is less than the threshold (from `HAND_TRACKING_CONSTANTS.PINCH_THRESHOLD`)
- Create function `isFingerExtended(landmarks: Landmark[], fingerType: 'middle' | 'ring' | 'pinky'): boolean` — determines if a finger is "open" (extended):
  - For each finger, calculate the angle at the PIP joint (second knuckle)
  - Map fingerType to landmark indices:
    - middle: MCP=9, PIP=10, TIP=12
    - ring: MCP=13, PIP=14, TIP=16
    - pinky: MCP=17, PIP=18, TIP=20
  - A finger is "extended" if its tip (y-coordinate) is significantly above (lower y-value, since y increases downward in screen space) its MCP joint. Use the threshold from `HAND_TRACKING_CONSTANTS.FINGER_OPEN_THRESHOLD`
  - Alternative approach: compare distance from wrist (landmark 0) to fingertip vs wrist to MCP. If tip is further from wrist than MCP, finger is extended
- Create function `areThreeFingersOpen(landmarks: Landmark[]): boolean` — returns true if middle, ring, AND pinky fingers are all extended
- Create function `areThreeFingersClosed(landmarks: Landmark[]): boolean` — returns true if middle, ring, AND pinky fingers are all NOT extended (curled)
- Create function `getPalmCenter(landmarks: Landmark[]): {x, y, z}` — average position of wrist (0), index MCP (5), middle MCP (9), ring MCP (13), pinky MCP (17)
- All functions must be pure (no side effects) and well-typed

### Step 1.2.2: Create Gesture Detector
**File:** `src/features/hand-tracking/utils/gesture-detector.ts` — CREATE
**Purpose:** Classifies the current hand configuration into one of the three gesture types: `'none'`, `'grab'`, or `'zoom'`
**Details:**
- Create function `detectGesture(landmarks: Landmark[]): HandGesture` that implements the following logic:
  1. Check if pinching (thumb + index tips are close together)
  2. If pinching AND other 3 fingers are OPEN → return `'grab'`
  3. If pinching AND other 3 fingers are CLOSED → return `'zoom'`
  4. If NOT pinching → return `'none'`
- The gesture detection must be **stable** — rapid flickering between states is a bad UX:
  - Implement a simple debounce: a gesture must be detected for at least 3 consecutive frames before it's confirmed
  - Create a `GestureStabilizer` class that tracks the last N gesture readings per hand and returns the most stable gesture:
    - Constructor takes `stabilityFrames: number` (default 3)
    - Method `update(gesture: HandGesture): HandGesture` — adds to history, returns the gesture only if it's been consistent for `stabilityFrames` frames; otherwise returns the previously stable gesture
  - Create two instances: one for left hand, one for right hand
- Edge case: if landmarks are partially occluded or low confidence, default to `'none'`

### Step 1.2.3: Integrate Gesture Detection into Hand Tracking Hook
**File:** `src/features/hand-tracking/hooks/use-hand-tracking.ts` — MODIFY
**Purpose:** Add gesture classification to the frame processing loop
**Details:**
- In the `requestAnimationFrame` loop, after extracting landmarks for each hand:
  1. Call `detectGesture(landmarks)` from the gesture detector (through the stabilizer)
  2. Update the `HandState.gesture` field with the classified gesture
  3. The store now includes gesture data alongside landmarks and position
- Initialize two `GestureStabilizer` instances (one per hand) as refs in the hook
- Ensure the gesture detection runs every frame that landmarks are processed — it's computationally cheap

## Verification Checklist
- [ ] Pinching thumb and index finger together while keeping other 3 fingers open → `gesture` field in store reads `'grab'`
- [ ] Pinching thumb and index finger together while curling other 3 fingers into fist → `gesture` field reads `'zoom'`
- [ ] Open hand (no pinch) → `gesture` field reads `'none'`
- [ ] Gesture transitions are stable — quickly waggling fingers doesn't cause rapid flickering between states (debounce works)
- [ ] Both hands can have independent gestures simultaneously (left hand grabbing, right hand zooming)
- [ ] Console logging the gesture state shows consistent updates at ~30fps
- [ ] `landmark-helpers.ts` functions are all pure — calling with the same input always returns the same output

## Rollback Plan
- Remove gesture detection from `use-hand-tracking.ts` — set gesture to `'none'` unconditionally
- Delete `gesture-detector.ts` and `landmark-helpers.ts`
- The app still tracks hands but without gesture classification

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Gesture always returns 'none' | Pinch threshold too small | Increase `PINCH_THRESHOLD` in constants. Debug by logging the actual thumb-index distance |
| Gesture flickers rapidly | Missing debounce/stabilizer | Verify `GestureStabilizer` is instantiated and `update()` is called, not bypassed |
| Finger extension detection is wrong | Y-axis convention mismatch | MediaPipe y-axis: 0 at top, 1 at bottom. An extended finger has a LOWER y-value at the tip than at the MCP. Verify this in your `isFingerExtended` function |
| Left/right hand gestures are swapped | MediaPipe handedness label is mirrored | MediaPipe labels "Right" for the hand that appears on the right side of the unmirrored image, which is actually the user's left hand if the image is mirrored. Apply the flip |
| Works for one hand but not two | Only processing first result | Ensure the loop iterates over all results and maps by handedness |
