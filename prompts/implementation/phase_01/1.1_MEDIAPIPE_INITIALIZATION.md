# Phase 1 — Step 1.1: MediaPipe Hand Tracking Initialization

## Context
This step integrates Google MediaPipe Hands into the application, enabling real-time webcam-based hand tracking. By the end, the app captures webcam video, processes it through MediaPipe, and outputs 21 hand landmarks per detected hand to the Zustand hand-tracking store. No gesture recognition yet — just raw landmark data flowing into the system.

## Skills to Load
- `3d-web-experience` — WebGL/camera integration patterns and performance considerations for real-time video processing alongside 3D rendering
- `research-engineer` — Ensure MediaPipe integration follows the latest API patterns (MediaPipe may have migrated to `@mediapipe/tasks-vision`)

## Prerequisites
- Phase 0 complete: Next.js running, R3F canvas rendering, both Zustand stores created
- MediaPipe packages installed (`@mediapipe/hands` or `@mediapipe/tasks-vision`)
- Webcam available on the device

## Instructions

### Step 1.1.1: Create MediaPipe Setup Utility
**File:** `src/features/hand-tracking/utils/mediapipe-setup.ts` — CREATE
**Purpose:** Encapsulates all MediaPipe initialization logic — loading the model, configuring detection parameters, and setting up the webcam stream
**Details:**
- Create an async function `initializeMediaPipe()` that:
  1. Requests webcam permission using `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'user', width: { ideal: 1280 }, height: { ideal: 720 } } })`
  2. Creates a `<video>` element (hidden, not displayed on screen) and sets its srcObject to the webcam stream
  3. Initializes MediaPipe Hands with configuration:
     - `maxNumHands: 2` (track both hands)
     - `modelComplexity: 1` (balance between speed and accuracy — 0 is fastest, 2 is most accurate)
     - `minDetectionConfidence: 0.7`
     - `minTrackingConfidence: 0.5`
  4. Returns an object containing: the video element, the MediaPipe Hands instance, and a cleanup function to release the webcam stream and dispose MediaPipe resources
- Create a function `processFrame(video: HTMLVideoElement, hands: MediaPipeHands)` that:
  1. Sends the current video frame to MediaPipe for processing
  2. Returns the results (hand landmarks, handedness — left/right classification)
- Handle errors gracefully:
  - Camera permission denied → set error in store: "Camera permission is required for hand tracking. Please allow camera access and reload."
  - Camera not found → "No camera detected. Please connect a webcam."
  - MediaPipe model failed to load → "Failed to load hand tracking model. Please check your internet connection."
- IMPORTANT: Check whether the project uses the new `@mediapipe/tasks-vision` API (recommended) or the legacy `@mediapipe/hands`. The tasks-vision API has a different initialization pattern using `HandLandmarker.createFromOptions()`. Prefer the newer API if available

### Step 1.1.2: Create the Hand Tracking Hook
**File:** `src/features/hand-tracking/hooks/use-hand-tracking.ts` — CREATE
**Purpose:** A React hook that manages the MediaPipe lifecycle — starts tracking on mount, processes frames in a `requestAnimationFrame` loop, and writes results to the hand-tracking store
**Details:**
- Call `initializeMediaPipe()` on mount inside a `useEffect`
- Once initialized, start a `requestAnimationFrame` loop that:
  1. Calls `processFrame()` to get hand landmark data
  2. For each detected hand, converts the raw MediaPipe landmarks (normalized 0-1 coordinates) into the `HandState` format:
     - `detected`: true
     - `landmarks`: array of 21 {x, y, z} points
     - `gesture`: 'none' for now (gesture recognition is Step 1.2)
     - `position`: calculate palm center as average of landmarks 0 (wrist), 5 (index MCP), and 17 (pinky MCP)
     - `velocity`: calculate by comparing current palm position to previous frame's palm position, divided by delta time
  3. Determines which result is the left hand and which is the right based on MediaPipe's handedness classification
  4. Writes to the hand-tracking store: `updateLeftHand(leftHandState)`, `updateRightHand(rightHandState)`
  5. If a hand is not detected, pass `null` to the update function
- On unmount, call the cleanup function to release webcam and dispose MediaPipe
- Return from the hook: `{ isReady: boolean, error: string | null }` — derived from the store
- PERFORMANCE: The RAF loop should target ~30fps for hand tracking (skip every other frame if needed) to leave headroom for the 3D rendering at 60fps. Use a frame counter or timestamp check to throttle

### Step 1.1.3: Create the Hand Tracking Provider Component

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Camera permission never prompts | HTTPS required or already denied | Run on `localhost` (allowed) or HTTPS. Check browser settings for blocked permissions |
| MediaPipe model fails to load | CDN blocked or network issue | Check if WASM files need to be served locally. Copy MediaPipe WASM assets to `public/` |
| Landmarks are mirrored | Webcam image is flipped | MediaPipe returns mirrored coordinates by default — flip x coordinates: `x = 1 - x` |
| Very high CPU usage (>80%) | Processing every frame at 60fps | Throttle to 30fps or reduce `modelComplexity` to 0 |
| "Cannot use import" error | Server-side rendering of browser APIs | Ensure all MediaPipe code is in `'use client'` components and uses dynamic imports |
