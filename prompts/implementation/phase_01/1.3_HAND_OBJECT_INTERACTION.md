# Phase 1 — Step 1.3: Hand-to-Object Interaction (Grab & Zoom)

## Context
With gesture recognition working (Step 1.2), this step connects gestures to actual 3D scene interactions. The **grab** gesture allows a user to pick up and move the spawned material objects, and the **zoom** gesture allows zooming the camera in/out. This completes the interaction model: hands detect → gestures classify → objects move or camera zooms.

## Skills to Load
- `3d-web-experience` — Mapping 2D screen-space hand coordinates to 3D world-space object positions, camera control patterns in R3F
- `react-patterns` — Managing imperative Three.js updates within React component lifecycle

## Prerequisites
- Step 1.2 complete: Gesture detection outputs `'grab'`, `'zoom'`, or `'none'` per hand
- Store has `leftPosition`, `rightPosition`, `zoomLevel` fields ready for updates
- MaterialMesh components read position from the store

## Instructions

### Step 1.3.1: Create Coordinate Mapping Utility
**File:** `src/features/hand-tracking/utils/coordinate-mapper.ts` — CREATE
**Purpose:** Converts normalized MediaPipe 2D coordinates (0-1 range) to Three.js 3D world coordinates
**Details:**
- Create function `screenToWorldPosition(normalizedX: number, normalizedY: number, camera: THREE.Camera, depth: number): THREE.Vector3`:
  - MediaPipe gives coordinates in [0, 1] range (top-left origin)
  - Convert to normalized device coordinates (NDC): x goes from -1 to 1, y goes from 1 to -1 (Three.js convention)
  - Use `THREE.Vector3.unproject(camera)` to project the 2D point into 3D space at the specified depth
  - The depth parameter controls how far from the camera the object is placed (use a fixed depth matching the spawn positions, typically z=0 on the scene plane)
- Create function `calculateRelativeVelocity(leftVelocity: {x, y, z}, rightVelocity: {x, y, z}): number`:
  - Computes the magnitude of the difference vector between left and right hand velocities
  - This represents how fast the user is "rubbing" the objects together
  - Returns a scalar value in world-space units per second
- Create function `normalizeToWorldBounds(position: [number, number, number], bounds: {minX, maxX, minY, maxY}): [number, number, number]`:
  - Clamps the position to keep objects within the visible scene area
  - Prevents objects from going off-screen when the hand moves to the edge of the webcam frame

### Step 1.3.2: Create Grab Interaction Hook
**File:** `src/features/hand-tracking/hooks/use-grab-interaction.ts` — CREATE
**Purpose:** When a hand's gesture is `'grab'`, this hook maps the hand's position to the corresponding object's position in 3D space
**Details:**
- Use `useFrame` from `@react-three/fiber` to run on every render frame (inside the R3F context)
- On each frame:
  1. Read left and right hand states from `useHandTrackingStore`
  2. If left hand gesture is `'grab'`:
     - Convert left hand's palm center (normalized coords) to 3D world position using `screenToWorldPosition`
     - Call `useSimulationStore.getState().updateLeftPosition(worldPos)` to move the left object
  3. If right hand gesture is `'grab'`:
     - Same for right hand → update right object position
  4. If a hand's gesture is NOT `'grab'` → do nothing (object stays where it was last placed)
- IMPORTANT: Use `useSimulationStore.getState()` for imperative updates inside `useFrame` — do NOT use the hook's return value inside the render loop (it would cause stale closures). The `getState()` pattern is the correct Zustand approach for frame-loop updates
- Add smooth interpolation (lerp) between the current position and target position to prevent jittery movement. Use `THREE.MathUtils.lerp` with a factor of 0.15-0.25 for responsive but smooth tracking

### Step 1.3.3: Create Zoom Interaction Hook
**File:** `src/features/hand-tracking/hooks/use-zoom-interaction.ts` — CREATE
**Purpose:** When a hand's gesture is `'zoom'`, this hook adjusts the camera's field of view based on the pinch distance
**Details:**
- Use `useFrame` from `@react-three/fiber`
- On each frame:
  1. Read hand states from `useHandTrackingStore`
  2. If ANY hand's gesture is `'zoom'`:
     - Record the initial pinch distance (thumb to index distance) when zoom first starts
     - On subsequent frames, compare current pinch distance to initial distance
     - If distance is increasing (fingers spreading apart within the pinch) → zoom IN (decrease FOV)
     - If distance is decreasing → zoom OUT (increase FOV)
     - Apply the zoom delta to `useSimulationStore.getState().updateZoomLevel(delta)`
  3. When zoom gesture ends (switches to 'none' or 'grab'):
     - The zoom level stays where it was (not reset)
     - Clear the initial pinch distance reference
  4. When the other 3 fingers open (gesture changes from 'zoom' to 'grab' or 'none') → zoom is cancelled, meaning the zoom operation stops but the current zoom level is preserved
- Access the Three.js camera via `useThree(state => state.camera)` and update its `fov` property
- Call `camera.updateProjectionMatrix()` after changing FOV
- Clamp FOV between 30° (zoomed in) and 120° (zoomed out)
- Disable OrbitControls zoom when hand tracking zoom is active to prevent conflicts

### Step 1.3.4: Integrate Interaction Hooks into Scene
**File:** `src/features/simulation/components/Scene.tsx` — MODIFY
**Purpose:** Add the grab and zoom hooks to the 3D scene
**Details:**
- Import and call `useGrabInteraction()` and `useZoomInteraction()` inside the Scene component
- These hooks must be called INSIDE the `<Canvas>` context because they use `useFrame` and `useThree`
- Remove or disable `OrbitControls` (used in Phase 0 for testing) — hand tracking now controls camera zoom. Keep OrbitControls for rotation only, with zoom disabled

### Step 1.3.5: Update MaterialMesh for Dynamic Positioning
**File:** `src/features/simulation/components/MaterialMesh.tsx` — MODIFY
**Purpose:** Smoothly update mesh position from the store
**Details:**
- Use `useFrame` to smoothly interpolate the mesh's actual position toward the target position from the store
- Read the target position via `useSimulationStore(state => state.leftPosition)` (or `rightPosition`)
- Apply `lerp` factor per frame: `mesh.current.position.lerp(new THREE.Vector3(...targetPos), 0.2)`
- This creates a smooth, slightly delayed "following" movement that feels natural and avoids jitter

## Troubleshooting
| Symptom | Likely Cause | Fix |
|---|---|---|
| Object jumps to wrong position | Coordinate mapping incorrect | Debug by logging normalized coords and world coords. Verify camera projection matrix is correct |
| Object movement is jittery | No interpolation or lerp factor too high | Reduce lerp factor to 0.1-0.15; check that `useFrame` delta time is being used consistently |
| Zoom doesn't work | FOV not calling `updateProjectionMatrix()` | Must call `camera.updateProjectionMatrix()` after changing `camera.fov` |
| Object follows the wrong hand | Left/right hand assignment swapped | MediaPipe handedness may be mirrored — verify which hand maps to which object |
| Both objects move with one hand | Not differentiating left vs right hand | Ensure left hand controls left object and right hand controls right object based on handedness from MediaPipe |
